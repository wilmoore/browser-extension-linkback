# Product Brief: Linkback

> Bidirectional backlinks for the browser. Keyboard-first.

## Problem

Modern work spans multiple "thinking surfaces" - inboxes (LinkedIn, Gmail, DMs), AI tools (ChatGPT, Claude), docs, notes, drafts, and research. Browsers treat pages as isolated. Humans don't.

Current workarounds create cognitive drag:
- Screenshots to shuttle context
- Copy/paste between tabs
- Manual notes and bookmarks
- Tab chaos

**Result:** Broken flow, lost context, friction in multi-surface workflows.

## Solution

A browser extension that creates a persistent relationship graph between URLs:
- Link any page to any other page instantly
- Auto-create inverse links (bidirectional by default)
- Recall and jump between related pages via Command+K palette
- Maintain a global inventory for cleanup

**Core rule:** If A links to B, then B links to A (always, automatically, atomically).

## Customer

Knowledge workers whose thinking spans multiple browser-based surfaces:
- **Inbox + AI assist:** LinkedIn thread ↔ ChatGPT convo
- **Research:** Article ↔ Notes ↔ Draft
- **Sales:** Prospect ↔ CRM ↔ Proposal doc
- **Personal:** Profile ↔ Messages ↔ Notes

Anyone who currently screenshots or copy/pastes to maintain context across pages.

## Primary UX Surfaces

### Jump Mode (Cmd+K)
Fast navigation while working:
- Command-palette list of links related to current page
- Search-as-you-type filtering
- Vim-style navigation: j/k (up/down), gg/G (top/bottom), / (search), Enter (open), Esc (close)

### Link Mode (Cmd+Shift+L)
Add a backlink with minimal friction:
- Tiny modal: "Link this page to..."
- URL prefilled from clipboard (editable)
- Title auto-filled (fetch or infer from hostname+path)
- On save: create A→B and B→A atomically

### Global Inventory (Cmd+Shift+K)
Review, audit, and clean the link graph:
- Left pane: searchable list of all stored pages (sort by most linked, recent, A-Z)
- Right pane: relationships for selected page (links from, links to)
- Actions: open, remove relationship, rename, merge duplicates, show orphans

## Key Behaviors

### Bidirectional Guarantee
One-way links are not supported. Link/unlink operations are symmetric.

### URL Normalization
- Strip common tracking params where safe
- Preserve essential params for apps that rely on them
- Manual "merge" available in Global Inventory

### Fast by Default
- Palette opens instantly (local cache)
- Cloud sync does not block UI

## Storage Architecture

### Cloud: Supabase (source of truth + cross-device sync)
- Auth: magic link or Google
- Data model: store relationship pairs per user, ensure uniqueness

### Local Cache (performance + offline)
- Cache per-URL linked list in extension storage (or IndexedDB)
- Queue failed writes as "pending sync", retry in background
- Allow link/jump locally when offline, reconcile on reconnect

## Design Principles

1. Keyboard-first
2. Vim-native navigation
3. Zero ceremony
4. Minimal surfaces (2 primary actions + 1 maintenance view)
5. Relationships over hierarchy
6. Fast > pretty
7. Privacy-respecting by default

## Risks

| Risk | Mitigation |
|------|------------|
| Storage reliability | Supabase + local cache + retry queue |
| URL fragility / duplication | Normalization + merge tools |
| Scope creep | Avoid tags/folders/dashboards; keep it link + jump + inventory |
| Privacy sensitivity | Treat link graph as sensitive; no third-party scripts; minimal permissions |

## Success Metrics

- Link in < 3 seconds
- Jump in < 1 second
- Find any stored relationship quickly via global inventory
- User stops screenshotting to shuttle context

**Signal it's working:** "My browser finally understands how I think."
